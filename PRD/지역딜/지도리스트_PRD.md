## 1. 개요

### 배경 및 문제 정의

현재 온라인 판매처 중심의 비대면 중개 방식은 구조적 한계가 있다:

- 기종당 1-2개 판매처만 노출 가능 → 다수 셀러 확보 불가
- 온라인 판매처의 재고/개통 한계 → 대량 개통 불가
- 통신사 가이드로 비대면 판매 정지 리스크
- 전국 단위 최저가 경쟁 → 셀러 이탈, 지속 불가능

### 해결 방향

지역 기반으로 오프라인 대리점을 노출하여:

- 다수 셀러를 지역별로 거의 무제한 노출
- 오프라인 대리점의 재고/개통 강점 활용
- 지역 단위 적정 경쟁으로 과열 방지
- 온라인 금지 시에도 대면 구매로 대응 가능

사용자에게는 주변 구들에 있는 소매대리점의 휴대폰 특가와 재고를 비교, 확인 가능하게 한다.

### 목표 및 성공 지표

**비즈니스 목표**

- 다수의 활성 셀러 확보를 통한 플랫폼 경쟁력 구축
- 지역 기반 적정 경쟁 구조로 셀러와 공존 가능한 지속가능한 플랫폼

**유저 목표**

- 내 주변 대리점들의 휴대폰 특가/재고를 한눈에 비교
- 가격이 저렴한 대리점을 쉽게 찾기

**측정 가능한 지표**

(미작성 - 선택사항)

## 2. 사용자 플로우

### 2.1 진입 경로

| 진입 경로 | 설명                                                                           |
| --------- | ------------------------------------------------------------------------------ |
| 모요 본체 | 휴대폰 찾기 탭 > 내주변 탭 클릭                                                |
| 외부 광고 | 광고 소재에 따라 다름. URL 파라미터에 지역/기기가 포함될 수도, 안 될 수도 있음 |

**외부 광고 예시**

| 광고 소재                       | URL 파라미터     | 진입 시 상태                             |
| ------------------------------- | ---------------- | ---------------------------------------- |
| "관악구 갤럭시S25가 4만원!"     | 지역/기기 포함   | 지역/기기 세팅됨, 지역 바텀시트 안 뜸    |
| "우리 동네 특가를 확인해보세요" | 지역/기기 미포함 | 일반 진입과 동일 (지역 설정 플로우 진행) |

### 2.2 지역 설정 플로우

**재방문 시 (쿠키에 구 저장된 경우)**

- 저장된 구로 자동 세팅 (바텀시트 없음)

**첫 방문 (쿠키에 구 저장 안 된 경우)**

- 시/구 선택 바텀시트 노출
- 시 선택 → 구 선택

**지역 재설정 클릭 시**

- 시/구 선택 바텀시트 다시 노출

**외부 광고 랜딩 (URL에 지역 포함된 경우)**

- URL 파라미터의 지역으로 자동 세팅

**외부 광고 랜딩 (URL에 지역 미포함된 경우)**

- 일반 진입과 동일하게 지역 설정 플로우 진행

**저장**

- 선택/세팅된 구 정보는 쿠키에 저장 → 재방문 시 유지

**권역(주변 구) 매핑**

- 지역 선택 시 해당 구만이 아니라 주변 구까지 함께 노출
- 서버에서 고정 매핑 관리 (예: 관악구 선택 시 → 관악구, 영등포구, 금천구 데이터 조회)
- 지역 선택할 때마다 의도적으로 3초 정도 로딩화면 노출
- 로딩 중 "관악구 및 그 주변 구를 보여드릴게요" 같은 안내 문구 노출

### 2.3 통신사 선택 플로우

**목적**

- 통신사 선택은 대리점 필터링이 아님
- 번호이동 가격 vs 기기변경 가격 중 어떤 가격을 보여줄지 결정하는 용도

**바텀시트 노출 시점**

- 첫 방문: 지역 설정 완료 직후 자동 노출
- 재방문 (쿠키에 통신사 저장된 경우): 노출 안 함
- 리스트 화면에서 "SKT 사용 중" 셀렉박스 클릭 시: 재노출

**선택지**

- SKT / KT / LG U+ / 알뜰폰

**저장**

- 선택한 통신사 정보는 쿠키에 저장 → 재방문 시 유지

### 2.4 기기/용량/색상 선택

**디폴트 값**

| 조건                                        | 디폴트 기기 | 용량  | 색상      |
| ------------------------------------------- | ----------- | ----- | --------- |
| OS가 Android 추정 또는 알 수 없음 또는 기타 | 갤럭시 S25  | 256GB | 색상 무관 |
| OS가 iOS 추정                               | 아이폰 17   | 256GB | 색상 무관 |

**변경 방법**

- 상단 기기/용량/색상/지역/통신사 셀렉박스 클릭하여 변경 가능
- 변경 시 지도/리스트 즉시 갱신 (별도 적용 버튼 없음)

**셀렉박스 옵션 범위**

- 기기: 오늘의 성지 제품에 뜨는 기기들과 동일
- 용량/색상: 선택한 기기에 존재하는 옵션만 노출

## 3. 화면 스펙: 지도

### 3.1 마커 표시 규칙

- 제휴 대리점 모두 마커로 표시
- 마커 정보: 통신사 로고 + 해당 대리점의 특가 가격(할부원금)
- 가격은 seller별로 선택된 기종*용량*현재 통신사에 맞는 spp 중 가장 할부원금이 싼 것
- 가격은 색상과 무관하게 동일 (동일 기기/용량이면 색상에 관계없이 가격 동일)
- 여러 마커가 겹치면 네이버 지도 API 기능으로 처리 (줌아웃 시 합쳐지고, 줌인 시 여러 개로 보임)

**셀러 노출 조건**

- 해당 권역에 위치를 두고 있고 & 선택된 기종\*용량에 대해 spp가 존재하면 노출
- 전 색상 품절인 매장도 노출됨 (단, [품절] 표시)

**마커 클릭 시**

- 해당 대리점의 해당 기기 특가 상세페이지로 이동
- 색상 "무관" 상태에서 클릭 시, 서버에서 내려주는 디폴트 색상으로 상세페이지 랜딩

### 3.2 지도 조작

**지도 기본 뷰**

- 디폴트 비율: 선택한 지역이 속한 권역의 대리점 마커들이 다 보이는 비율

**조작 방법**

- 줌인, 줌아웃, 드래그로 이동 가능
- 지도를 조금이라도 움직이면 [이 지역 재검색] 플로팅 버튼 노출

**이 지역 재검색 버튼 클릭 시**

- 지도 뷰에 마커 새로 갱신
- 리스트도 갱신
- 상단 구 이름을 [현재 지도]로 변경

**[현재 지도] 상태에서 구 셀렉박스로 다른 구 선택 시**

- 해당 구 권역으로 다시 포커싱

### 3.3 지도-리스트 싱크

- 지도와 리스트는 완벽히 1:1 매핑
- 지도 현재 뷰포트에 보이는 마커 = 리스트에 노출되는 셀러
- 예: 디폴트 비율에서 30개 대리점이 보였는데 특정 지역으로 줌인 → [이 지역 재검색] 클릭 → 지도는 새 뷰포트 기준으로 마커 갱신, 리스트도 새 뷰포트에 보이는 매장들만 갱신
- 갱신 트리거: (1) 바텀시트에서 지역값 변경 (2) 이 지역 재검색 버튼 클릭

### 3.4 지도 포커스 모드

**진입**

- 지도 터치 시

**동작**

- 리스트가 내려가고 지도가 화면 대부분 차지
- 리스트는 핸들러만 보이는 정도로 내려감

**해제**

- 리스트 핸들러를 위로 드래그

## 4. 화면 스펙: 리스트

### 4.1 레이아웃

**Initial state**

- 상단 섹션 + 지도 + 리스트 1개가 보임

**리스트 드래그 시**

- 지도 위를 덮으면서 올라옴
- 일정 수준 이상 스크롤하면 상단 탭(특가/내주변/오성)이 사라지고, 기종 선택 박스와 필터 영역은 top에 sticky

**노출 조건**

- 선택한 구가 속한 권역에 매장이 1개 이상 있는 경우

### 4.2 리스트 아이템 정보

| 항목        | 설명                                                |
| ----------- | --------------------------------------------------- |
| 통신사 로고 | 해당 대리점의 통신사                                |
| 대리점 이름 | -                                                   |
| 접근성      | 주요 역에서 도보 n분 (예: 신림역에서 도보 5분)      |
| 가격        | 선택된 기기/용량에 대한 해당 대리점의 특가 할부원금 |

### 4.3 정렬 및 필터

**정렬 기준**

- 1차: 가격순 (할부원금 낮은 순)
- 2차: 대리점 이름 ㄱㄴㄷ순 (할부원금 동일 시)
- 품절 대리점도 가격 기준으로 중간에 섞여서 노출됨

**필터**

| 필터        | 동작                                                                                                                          |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------- |
| 통신사 전체 | 클릭 시 통신사 선택 바텀시트 노출. 특정 통신사(예: SKT) 선택 시 해당 통신사 대리점만 필터링되어 노출. 지도 마커도 함께 갱신됨 |
| 품절 제외   | 현재 선택된 옵션(기기/용량/색상)에 대해 품절인 대리점을 필터링하여 제외. 지도 마커도 함께 갱신됨                              |

※ 이 필터의 통신사 선택은 **대리점 필터링** 용도임. 상단 "SKT 사용 중" 셀렉박스의 통신사 선택(번이/기변 가격 결정용)과는 별개.

**필터 상태 유지**

- 지역 변경 시: 필터 유지됨
- 세션 내: 필터 유지됨 (탭 이동, 페이지 이동 후 복귀 시에도 유지)
- 세션 종료 시: 필터 초기화 (브라우저 저장소에 저장하지 않음)

### 4.4 품절 표시

- 재고는 대리점별로 색상 단위로 관리됨 (예: 블랙 4개, 화이트 3개)
- 색상 "무관" 선택 시: 해당 기기의 모든 색상이 품절인 대리점만 [품절] 표시
- 특정 색상 선택 시: 해당 색상 재고 없는 대리점은 [품절] 표시

### 4.5 페이지네이션

- 무한스크롤, 약 10개씩 로드

**리스트 아이템 클릭 시**

- 해당 대리점의 해당 기기 특가 상세페이지로 이동 (마커 클릭과 동일)

## 5. 예외 케이스

### 5.1 매장 없는 경우

권역 내 매장이 0개일 때:

1. "특가를 보여드릴 대리점이 없어요" 메시지 노출
2. "지역 재설정" 링크 노출
3. 하단에 "전국 배송 가능한 온라인 특가는 어떠세요?" 대안 섹션 노출

### 5.3 유효하지 않은 URL

- URL 파라미터 지역이 유효하지 않음 (존재하지 않는 구, 서비스 미지원 지역)
- "접근이 불가능합니다" 에러 다이얼로그 → 버튼 클릭 시 모요 홈으로 이동

---

### 5.4 상세 진입 시 현재 통신사 검증

**현재 통신사를 모르는 채로 특가 상세에 바로 진입하는 경우**

상세 진입 시 URL의 sppID 기준으로 배경 화면을 먼저 그린 후, 다음 로직 수행:

1. 브라우저 저장소에 현재통신사 값이 없음 → 현재통신사 선택 바텀시트 노출
2. 브라우저 저장소에 현재통신사 값이 있음 → URL 파라미터의 현재통신사값과 비교
   - 불일치 → 현재통신사 선택 바텀시트 노출
   - 일치 → 해당 sppID와 호환 여부 체크
     - 호환됨 → 정상 진행
     - 호환 안됨 → 안내 다이얼로그 노출

**호환 안 되는 케이스**

| SPP 가입유형 | 조건                          | 안내 문구                                            |
| ------------ | ----------------------------- | ---------------------------------------------------- |
| 번호이동     | 유저 현재 통신사 = SPP 통신사 | "[통신사] 이용자는 해당 특가를 이용할 수 없어요"     |
| 기기변경     | 유저 현재 통신사 ≠ SPP 통신사 | "현재 ([유저 통신사]) 쓰시는 분들만 가능한 특가에요" |

- [특가 보러가기] 클릭 시 → 내주변 찾기 첫화면으로 이동

### 5.5 종료/비활성화된 특가 처리

**상세 페이지 진입 시점에 이미 종료된 특가**

- 안내 문구: "앗, 종료된 특가에요 / 다른 특가를 신청해주세요"
- [특가 보러가기] 클릭 시 → 내주변 찾기 첫화면으로 이동

**방문 예약 버튼 클릭 시 validation 결과 종료된 경우**

- 안내 문구: "앗, 종료된 특가에요 / 조건을 변경하거나 다른 대리점 특가를 신청해주세요"
- [닫기] 클릭 시 → 다이얼로그만 닫힘 (상세 페이지에서 조건 변경 가능)

## 6. URL 및 상태 관리

### 6.1 URL 형식 및 파라미터

**URL 형식**

`/phones/local-deal?currentNetworkOperator={}&skuCode={}&skuStorage={}&color={}&legalDongCode={}`

**파라미터 설명**

| 파라미터               | 설명                              |
| ---------------------- | --------------------------------- |
| currentNetworkOperator | 현재 통신사 (SKT, KT, LGU+, MVNO) |
| skuCode                | 기기 코드                         |
| skuStorage             | 용량                              |
| color                  | 색상                              |
| legalDongCode          | 법정동코드 (구 정보)              |

### 6.2 디폴트값 처리

**기기/용량**

- 파라미터가 비어있는 URL로 진입 시 디폴트값으로 채움
- 디폴트: OS가 Android면 갤럭시S25, iOS면 아이폰17 / 용량 256GB
- skuCode만 있고 skuStorage가 없는 경우: URL의 skuCode 사용 + 해당 기기의 최소 용량으로 자동 선택

**현재통신사/구**

- 파라미터가 비어있는 URL로 진입 시, 배경 UI 렌더링을 위해 내부 state에 임시값(SKT/관악구)을 세팅함
- 이 임시값은 UI 렌더링 전용이며, URL 파라미터나 브라우저 저장소(쿠키)에는 반영하지 않음
- 사용자가 바텀시트에서 직접 선택을 완료한 시점에만 해당 값을 URL 파라미터와 브라우저 저장소에 저장함
- 바텀시트 노출은 **무조건** 다음 순서로 진행:
  1. 구 선택 바텀시트 노출
  2. 현재 통신사 선택 바텀시트

### 6.3 브라우저 저장소 연동

**저장 항목**

- 선택한 구 정보 → 쿠키에 저장
- 선택한 통신사 정보 → 쿠키에 저장

**저장 안 함**

- 필터 상태 (재방문 시 초기화)

### 6.4 URL vs 저장소 충돌 처리

**현재 통신사 (currentNetworkOperator) 처리**

- URL의 통신사값과 브라우저 저장소의 값을 대조
- 두 값이 다를 경우: 현재 통신사 선택 바텀시트 노출
- 예: URL에 currentNetworkOperator=skt로 진입했는데, 브라우저 저장소에 통신사값이 null이면 (직접 입력한 적 없음) → 통신사 선택 바텀시트 노출

**법정동코드 (legalDongCode) 처리**

| 케이스                       | 처리                                                                                                          |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------- |
| URL에 법정동코드가 있는 경우 | 브라우저 저장소값과 다르더라도 지역 선택 바텀시트 안 띄움. URL의 지역으로 바로 지도&리스트 노출               |
| URL에 법정동코드가 없는 경우 | 배경 UI를 그리기 위해 임시로 관악구를 세팅. 그 위에 **무조건** 구 선택 바텀시트를 노출하여 사용자 입력을 받음 |

## 8. 화면 상태 정의

### 초기 상태 (진입 시)

- 디폴트값으로 지도&리스트가 그려진 상태로 시작
- 디폴트값: OS에 따라 갤럭시S25 또는 아이폰17 / 256GB / 색상 무관 / SKT 사용 중 / 관악구
- 그 위에 지역 선택 바텀시트가 노출됨 (쿠키에 지역 정보가 없는 경우)

### 바텀시트 노출 상태

- 바텀시트 뒤 지도/리스트는 딤 처리
- 종류: 지역 선택 바텀시트, 통신사 선택 바텀시트, 기기/용량/색상 선택 바텀시트, 리스트 필터용 통신사 바텀시트

### 지역 선택 후 로딩 상태

- 3초간 로딩 화면 노출
- 안내 문구: "관악구 및 그 주변 구를 보여드릴게요" 등

### 데이터 로딩 상태

- 기기/용량/색상 변경, 필터 적용, 이 지역 재검색 등 API 호출 시
- 리스트 영역만 스켈레톤 처리

### 지도+리스트 기본 상태

- 상단 섹션 + 지도 + 리스트 1개가 보이는 상태
- 지도에 권역 내 대리점 마커 노출
- 리스트에 동일한 대리점 목록 노출

### 지도 포커스 모드

- 지도 터치 시 진입
- 리스트가 내려가고 지도가 화면 대부분 차지 (핸들러만 보임)
- 리스트 핸들러를 위로 드래그하면 기본 상태로 복귀

### 리스트 확장 상태

- 리스트를 위로 드래그하면 지도를 덮으면서 올라옴
- 일정 수준 이상 스크롤 시 상단 탭(특가/내주변/오성) 사라지고, 기종 선택 박스와 필터 영역은 top에 sticky

### 매장 없음 상태

- 권역 내 매장 0개일 때
- "특가를 보여드릴 대리점이 없어요" 메시지
- "지역 재설정" 링크
- 하단에 "전국 배송 가능한 온라인 특가는 어떠세요?" 대안 섹션

### 에러 상태

- URL 파라미터 지역이 유효하지 않은 경우
- "접근이 불가능합니다" 에러 다이얼로그 → 버튼 클릭 시 모요 홈으로 이동
