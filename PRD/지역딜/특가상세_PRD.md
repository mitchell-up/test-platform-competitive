## 1. 개요

### 배경 및 문제 정의

지도/리스트 페이지에서 사용자가 관심 있는 대리점의 특가를 발견했을 때, 해당 특가의 상세 정보(가격 구성, 조건, 매장 정보 등)를 확인하고 방문 예약까지 이어질 수 있는 페이지가 필요하다.

현재 문제점:

- 리스트에서는 할부원금만 보여서, 실제 총 비용이나 혜택 구조를 알기 어려움
- 대리점 정보(위치, 영업시간 등)를 확인할 수 없음
- 관심 있는 특가를 발견해도 다음 행동(방문 예약)으로 연결되지 않음

### 해결 방향

특가 상세 페이지를 통해:

- 특가의 전체 가격 구성(공시지원금, 추가지원금, 할부원금 등)을 투명하게 제공
- 대리점 정보(위치, 영업시간, 연락처)를 한눈에 확인
- 방문 예약 기능으로 사용자의 다음 행동을 자연스럽게 유도

### 목표 및 성공 지표

**비즈니스 목표**

- 특가 상세 페이지 → 방문 예약 전환율 극대화
- 대리점-사용자 간 연결(리드) 생성

**유저 목표**

- 특가의 실제 비용과 혜택을 명확히 이해
- 대리점 방문 전 필요한 정보를 모두 확인
- 간편하게 방문 예약 완료

**측정 가능한 지표**

| 지표 | 목표 |
| --- | --- |
| 상세 페이지 → 방문 예약 전환율 | 10% 이상 |
| 방문 예약 완료율 | 80% 이상 |
| 상세 페이지 이탈률 | 50% 이하 |

## 2. 사용자 플로우

### 2.1 진입 경로

| 진입 경로 | 설명 |
| --- | --- |
| 지도 마커 클릭 | 지도에서 대리점 마커 클릭 시 해당 대리점의 특가 상세로 이동 |
| 리스트 아이템 클릭 | 리스트에서 대리점 카드 클릭 시 해당 대리점의 특가 상세로 이동 |
| 외부 링크/공유 | 특가 상세 URL을 통한 직접 진입 |

### 2.2 상세 페이지 내 플로우

```
상세 페이지 진입
    ↓
가격 정보 확인 (공시지원금, 추가지원금, 할부원금 등)
    ↓
옵션 변경 (선택사항: 용량, 색상 변경)
    ↓
대리점 정보 확인 (위치, 영업시간, 연락처)
    ↓
방문 예약 버튼 클릭
    ↓
예약 정보 입력 (이름, 연락처, 방문 희망 시간)
    ↓
예약 완료
```

### 2.3 옵션 변경 플로우

**용량/색상 변경 시**

- 상세 페이지 내에서 용량/색상 셀렉박스로 변경 가능
- 변경 시 가격 정보가 실시간으로 갱신됨
- 해당 옵션이 품절인 경우 품절 표시와 함께 선택 불가

**통신사/기기 변경 시**

- 상세 페이지에서는 변경 불가
- 다른 기기/통신사를 보려면 리스트 페이지로 돌아가야 함

## 3. 화면 스펙: 특가 상세

### 3.1 상단 영역

**헤더**

- 뒤로가기 버튼 (리스트 페이지로 복귀)
- 공유 버튼

**기기 정보 카드**

| 항목 | 설명 |
| --- | --- |
| 기기 이미지 | 선택된 색상의 기기 이미지 |
| 기기명 | 예: 갤럭시 S26 Ultra |
| 용량/색상 | 예: 256GB / 티타늄 블랙 |
| 용량 셀렉박스 | 해당 기기의 가능한 용량 옵션 |
| 색상 셀렉박스 | 해당 기기의 가능한 색상 옵션 |

### 3.2 가격 정보 영역

**가격 구성 테이블**

| 항목 | 설명 |
| --- | --- |
| 출고가 | 기기 정가 |
| 공시지원금 | 통신사 공식 지원금 |
| 추가지원금 | 대리점 추가 지원금 |
| 할부원금 | 출고가 - 공시지원금 - 추가지원금 |
| 월 할부금 | 24개월 기준 월 납부액 (무이자 할부 기준) |

**요금제 정보**

| 항목 | 설명 |
| --- | --- |
| 적용 요금제 | 해당 특가에 필요한 요금제 |
| 월 요금 | 요금제 월 납부액 |
| 약정 기간 | 24개월/36개월 등 |

**가입 유형 표시**

- 번호이동 / 기기변경 여부 명시
- 현재 사용자의 통신사와 매칭되는지 표시

### 3.3 대리점 정보 영역

| 항목 | 설명 |
| --- | --- |
| 대리점명 | 예: 모바일마트 신림점 |
| 통신사 | SKT / KT / LG U+ |
| 주소 | 상세 주소 |
| 접근성 | 주요 역에서 도보 n분 (예: 신림역에서 도보 5분) |
| 영업시간 | 평일/주말 영업시간 |
| 연락처 | 전화번호 (클릭 시 전화 연결) |

**지도 미니맵**

- 대리점 위치를 보여주는 작은 지도
- 클릭 시 네이버/카카오 지도 앱으로 연결

### 3.4 하단 고정 영역

**방문 예약 버튼**

- 화면 하단에 고정
- 버튼 텍스트: "방문 예약하기"
- 품절 시: 비활성화 + "품절" 텍스트

## 4. 화면 스펙: 방문 예약

### 4.1 예약 바텀시트

**노출 시점**

- "방문 예약하기" 버튼 클릭 시

**입력 필드**

| 필드 | 필수 여부 | 설명 |
| --- | --- | --- |
| 이름 | 필수 | 예약자 이름 |
| 연락처 | 필수 | 휴대폰 번호 (자동 하이픈) |
| 방문 희망일 | 필수 | 날짜 선택 (오늘 ~ 7일 후) |
| 방문 희망 시간 | 필수 | 시간대 선택 (영업시간 내) |
| 요청사항 | 선택 | 자유 텍스트 입력 |

**버튼**

- "예약 완료" 버튼
- 필수 필드 미입력 시 비활성화

### 4.2 예약 완료 화면

**완료 메시지**

- "방문 예약이 완료되었어요!"
- 예약 정보 요약 (대리점명, 방문일시, 특가 정보)

**안내 사항**

- "대리점에서 확인 후 연락드릴 예정이에요"
- "예약 내용은 카카오톡/문자로 발송됩니다"

**버튼**

- "확인" → 리스트 페이지로 이동

## 5. 예외 케이스

### 5.1 품절 처리

**특정 색상 품절**

- 해당 색상 옵션에 [품절] 뱃지 표시
- 선택 불가 (회색 처리)
- 다른 색상은 정상 선택 가능

**전 색상 품절**

- 방문 예약 버튼 비활성화
- "현재 재고가 없어요. 다른 대리점을 확인해보세요" 안내
- "다른 대리점 보기" 링크 제공

### 5.2 종료/비활성화된 특가

**상세 페이지 진입 시 이미 종료된 특가**

- 안내 문구: "앗, 종료된 특가에요 / 다른 특가를 신청해주세요"
- [특가 보러가기] 클릭 시 → 내주변 찾기 첫화면으로 이동

**방문 예약 버튼 클릭 시 validation 결과 종료된 경우**

- 안내 문구: "앗, 종료된 특가에요 / 조건을 변경하거나 다른 대리점 특가를 신청해주세요"
- [닫기] 클릭 시 → 다이얼로그만 닫힘 (상세 페이지에서 조건 변경 가능)

### 5.3 현재 통신사 불일치

**번호이동 특가인데 유저가 해당 통신사 사용 중인 경우**

- 안내 문구: "[통신사] 이용자는 해당 특가를 이용할 수 없어요"
- [다른 특가 보기] 클릭 시 → 리스트 페이지로 이동

**기기변경 특가인데 유저가 다른 통신사 사용 중인 경우**

- 안내 문구: "현재 ([유저 통신사]) 쓰시는 분들만 가능한 특가에요"
- [다른 특가 보기] 클릭 시 → 리스트 페이지로 이동

### 5.4 유효하지 않은 URL

- sppID가 존재하지 않거나 유효하지 않은 경우
- "접근이 불가능합니다" 에러 다이얼로그
- 버튼 클릭 시 모요 홈으로 이동

### 5.5 예약 실패

**네트워크 오류**

- "일시적인 오류가 발생했어요. 다시 시도해주세요"
- [다시 시도] 버튼

**대리점 예약 불가 상태 (임시 휴무 등)**

- "현재 예약을 받지 않는 대리점이에요"
- "직접 연락해보세요" + 전화번호 표시

## 6. URL 및 상태 관리

### 6.1 URL 형식

**특가 상세 URL**

```
/phones/local-deal/detail?sppId={}&currentNetworkOperator={}&skuStorage={}&color={}
```

**파라미터 설명**

| 파라미터 | 필수 | 설명 |
| --- | --- | --- |
| sppId | 필수 | 특가 고유 ID (대리점 + 기기 + 가입유형 조합) |
| currentNetworkOperator | 선택 | 현재 통신사 (SKT, KT, LGU+, MVNO) |
| skuStorage | 선택 | 용량 (미지정 시 최소 용량) |
| color | 선택 | 색상 (미지정 시 디폴트 색상) |

### 6.2 디폴트값 처리

**용량 (skuStorage)**

- URL에 없는 경우: 해당 기기의 최소 용량으로 자동 선택

**색상 (color)**

- URL에 없는 경우: 서버에서 내려주는 디폴트 색상으로 선택

**현재 통신사 (currentNetworkOperator)**

- URL에 없고, 브라우저 저장소에도 없는 경우: 통신사 선택 바텀시트 노출
- URL에 있는 경우: URL 값과 브라우저 저장소 값 비교 후 불일치 시 바텀시트 노출

### 6.3 상태 변경 시 URL 업데이트

- 용량/색상 변경 시 URL 파라미터 실시간 업데이트 (히스토리 push)
- 뒤로가기 시 이전 옵션 상태로 복원됨

## 7. 데이터 정책

### 특가 데이터

- sppID 기준으로 특가 정보 조회
- 가격, 요금제, 재고 정보 포함

### 대리점 데이터

- 대리점 기본 정보 (이름, 주소, 연락처, 영업시간)
- 위치 좌표 (지도 표시용)

### 예약 데이터

- 예약 생성 시 서버에 저장
- 예약자 정보 + 특가 정보 + 방문 희망 시간
- 대리점에 알림 전송

## 8. 화면 상태 정의

### 로딩 상태

- 상세 페이지 진입 시 스켈레톤 UI 노출
- 가격 정보, 대리점 정보 영역 각각 스켈레톤

### 기본 상태

- 모든 정보가 정상 로드된 상태
- 방문 예약 버튼 활성화

### 품절 상태

- 선택된 옵션이 품절인 경우
- 방문 예약 버튼 비활성화 + "품절" 표시

### 옵션 변경 중 상태

- 용량/색상 변경 시 가격 영역만 로딩 표시
- 나머지 영역은 유지

### 예약 진행 상태

- 바텀시트 오픈
- 뒤 화면 딤 처리

### 에러 상태

- 특가 종료, 통신사 불일치, 유효하지 않은 URL 등
- 에러 다이얼로그 노출
